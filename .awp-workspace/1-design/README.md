# Design

Use this folder for design intent, maturity tracking, and delivery sequencing.

This `README.md` is the human stage guide.
The uppercase files below are the canonical design-state artifacts.

Core artifacts:
- `PROJECT_BRIEF.md`
- `DESIGN_STATES.md`
- `TASK_READINESS.md` — per-task IRG scores, blocking unknowns, and build admission
- `ROADMAP.md`

Stage templates:
- `templates/PROJECT_BRIEF_TEMPLATE.md`
- `templates/ADR_TEMPLATE.md`

Common additions in real projects:
- `QUALITY_REQUIREMENTS.md` for durable nonfunctional or cross-cutting requirements that do not need a formal specialist advisor gate
- feature spec documents
- interface or contract notes
- architecture decision records in `decisions/`
- active specialist advisor requests when advisor input is needed to shape the design
