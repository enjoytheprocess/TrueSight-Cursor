# Examples

Committed example workspaces showing what a real initialized workspace looks like.

Browse these to understand the register structure and state conventions before starting your own workspace. Read `docs/guides/worked-example.md` for the narrative walkthrough that explains how each state was reached.

## snippets-api

A single-component workspace for a small REST API (create and retrieve code snippets).

**State:** two complete loops finished, a third in progress.
- `SETUP-001` — done (workspace bootstrapped, code repo connected)
- `TASK-001` — done (POST /snippets + GET /snippets/:id, SQLite storage)
- `TASK-002` — design phase, `needs_detail` (deletion endpoint, semantics unresolved)

**Note:** the generated `.md` table views in this example are static snapshots. In a live workspace they are regenerated by `make render` after every `.yaml` edit.
