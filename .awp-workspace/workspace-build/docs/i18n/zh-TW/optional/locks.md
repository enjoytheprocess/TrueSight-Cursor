# 鎖定（Locks）

**觸發條件**：當多個代理或貢獻者在同一組檔案上並行建置時添加此模組。若沒有鎖定，對共享登記表（WORK_QUEUE、DESIGN_STATES、規格檔案）的並行寫入會產生難以解決的衝突。

## 檔案

| 檔案 | 角色 |
|------|------|
| `2-build/LOCKS.yaml` | 有效和已釋放的鎖定記錄 |
| `2-build/LOCKS.md` | 生成的視圖（不直接編輯；使用 `make render`） |

## 運作方式

規律很簡單：**在編輯前聲明，合併或交接後立即釋放。**

每個鎖定記錄包含：
- `status` — `active`（活躍）| `released`（已釋放）
- 被鎖定的檔案或檔案集
- 持有鎖定的任務 ID 和所有者
- 聲明和釋放的時間戳記

當任務在 WORK_QUEUE 中進入 `in_progress` 時，在 WORK_QUEUE 記錄中記錄對應的鎖定 ID。當任務移至 `awaiting_human_review` 或被交接時，釋放鎖定。

不要跨階段或在阻塞/返工期間持有鎖定——釋放它們，在工作恢復時重新聲明。

## 與迴圈的連接

鎖定是建置（Build）階段的協調工具。它們不取代同步（Sync）分類——它們只防止使分類更困難的寫入衝突。對於需要超過鎖定單獨能提供的更多協調的更高並發專案（例如並行的設計（Design）和建置（Build）階段，或多代理執行），見 `docs/optional/concurrency-overlay.md`。
