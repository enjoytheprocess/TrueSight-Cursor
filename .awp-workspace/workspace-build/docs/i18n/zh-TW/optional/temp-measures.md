# 臨時措施（Temporary Measures）

**觸發條件**：當臨時例外或戰術性變通方案需要追蹤的移除目標時添加此模組——例如：跳過的驗證規則、硬編碼的值、有已知到期日的功能旗標，或因等待上游修復而固定在已知問題版本的依賴。

## 檔案

| 檔案 | 角色 |
|------|------|
| `2-build/TEMP_MEASURES.yaml` | 有效的措施（open / removal_in_progress） |
| `2-build/archive/TEMP_MEASURES.yaml` | 已關閉的措施 |

## 運作方式

每個臨時措施需要三件事：
1. **所有者** — 負責移除的具名人員或角色
2. **退出觸發條件** — 使移除適當的條件（日期、上游發布、任務完成）
3. **移除目標** — 日期或連結的任務 ID

`status: removal_in_progress` 的措施在移除確認之前保留在有效檔案中。一旦移除，在同一個同步（Sync）週期中設定 `status: closed`，並將記錄移至封存檔案。

`linked_tasks` 記錄哪些任務建立、依賴或將移除此措施——這是在相關任務進行中時，在建置（Build）期間浮現臨時措施的主要方式。

## 與迴圈的連接

在建置開始時：檢查連結至當前任務的臨時措施。若措施的退出觸發條件已滿足，將其移除記錄為實作的一部分，並設定為 `removal_in_progress`。同步（Sync）在驗證（Verify）接受任務後確認關閉。
