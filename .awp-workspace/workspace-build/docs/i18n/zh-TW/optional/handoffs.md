# 交接（Handoffs）

**觸發條件**：當任務所有權在人員或代理之間轉移，且轉移歷史記錄很重要時添加此模組——例如：人類接管代理留下部分完成的任務時，代理恢復人類開始的工作時，或在生命週期階段邊界跨團隊成員的責任轉移時。

## 檔案

`4-sync/HANDOFFS.md` — 只能追加的日誌；每次轉移一個記錄。

## 運作方式

每當任務更換所有者，或以需要其他人接手的方式轉換生命週期階段時，添加一個記錄。每個記錄包含：
- **日期**
- **任務 ID**
- **從 / 至** — 誰或哪個代理在交接，誰在接手
- **階段轉換** — 例如 `in_progress → awaiting_human_review`
- **摘要** — 完成了什麼，還剩什麼
- **已變更的檔案** — 離開所有者的階段中觸碰的檔案
- **已執行的驗證** — 交接前檢查了什麼
- **開放的風險/後續事項** — 接手所有者需要知道的任何事情

日誌是只能追加的。不要編輯過去的記錄——如果交接摘要不完整，添加一個引用原始記錄的修正記錄。

## 與迴圈的連接

交接通常在任務在所有者之間移動時，在驗證（Verify）或同步（Sync）階段撰寫。接手的所有者在恢復建置（Build）或做出接受/拒絕決策之前讀取交接記錄。交接日誌不由同步（Sync）分類——它只是歷史記錄。
