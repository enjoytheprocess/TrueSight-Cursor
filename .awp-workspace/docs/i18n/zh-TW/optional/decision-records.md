# 決策記錄（Decision Records）

**觸發條件**：當架構、合約、所有權或推出決策需要持久記錄時添加此模組。用於後果重大的決策——難以撤回或未來貢獻者需要理解以避免重新爭論的決策。

## 檔案

| 檔案 | 角色 |
|------|------|
| `1-design/decisions/` | 每個決策一個 Markdown 檔案；無規定命名——使用簡短的 slug（例如 `auth-token-storage.md`） |

## 運作方式

每個決策記錄回答四個問題：
1. **決定了什麼？** — 具體的技術或架構選擇
2. **為什麼？** — 驅動決策的限制、取捨和背景
3. **拒絕了哪些替代方案？** — 以及原因
4. **後果是什麼？** — 包括已知的風險或後續義務

決策記錄由人類撰寫；代理可以起草供審查，但不得單方面最終確定記錄。一旦撰寫，記錄是只能追加的——如果決策被撤銷，添加一個引用舊記錄的新記錄，而非編輯它。

## 與迴圈的連接

在准入時，在 `TASK_READINESS.yaml` 的 `decision_links` 欄位和對應的 `WORK_QUEUE.yaml` 記錄中引用相關的決策記錄。代理在建置開始時讀取這些連結，在實作之前理解具有約束力的限制。決策記錄不是設計規格——它記錄的是「為什麼」，而非「什麼」。
